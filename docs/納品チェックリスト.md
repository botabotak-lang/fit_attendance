# 勤怠アプリ（fit-attendance）納品チェックリスト

## 1. Supabase

1. プロジェクトを用意し、[supabase/fit_attendance.sql](../supabase/fit_attendance.sql) を **SQL Editor** で1回実行する。
2. **Project Settings → API** から `URL` と `service_role` キーをコピーする（`service_role` は絶対にクライアントや Git に載せない）。

## 2. Vercel

1. GitHub リポジトリを Vercel にインポートする。
2. **Environment Variables** に以下を設定する。
   - `NEXT_PUBLIC_SUPABASE_URL`
   - `SUPABASE_SERVICE_ROLE_KEY`
   - （推奨）`ATTENDANCE_API_KEY` と `NEXT_PUBLIC_ATTENDANCE_API_KEY` を **同一の長いランダム文字列** にする。
   - （推奨）`ADMIN_BASIC_AUTH` を `ユーザー名:パスワード` にする（`/admin` のみ Basic 認証）。
3. 本番 URL を FIT に共有する。

## 3. 運用メモ（FIT 向け）

- **打刻データ**は Supabase に保存され、複数の端末・ブラウザから同じデータを参照できる。
- **社員マスタ**の変更は `/admin` 画面の最下部から行う（社員マスタ保存後、打刻画面は自動で再読込される。別タブは最大約1分またはフォーカスで更新）。
- **旧 localStorage データ**は、本番に初めてアクセスしたタブで1回だけ自動アップロードを試みる（成功後はブラウザ内の旧キーを削除）。
- `/admin` に Basic 認証を付けていない場合、**URL を知っている人は誰でも**管理画面に入れる。必ず `ADMIN_BASIC_AUTH` か Vercel の **Deployment Protection** のいずれかを使うこと。
